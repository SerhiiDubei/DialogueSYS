# Демо "4 друзі в барі" - Dialogue Manager 3

## Опис
Мінімальна демо-сцена з 4 персонажами, яка демонструє:
- Ініціалізацію параметрів героя
- Умовні діалоги (L1/L2)
- Хаб-меню з поверненням
- Систему згоди персонажів

## Структура файлів

- `res://scripts/game_state.gd` - стан гри (4 параметри + прапорці)
- `res://scripts/demo_hub.gd` - логіка запуску діалогу
- `res://scenes/DemoHub.tscn` - головна сцена
- `res://dialogue/demo.dialogue` - весь діалог

## Як запустити

1. Відкрийте проект в Godot 4.3+
2. Переконайтеся, що плагін Dialogue Manager увімкнено (Project → Project Settings → Plugins)
3. Натисніть F5 або Play

## Як працює

1. **Старт** → коротке інтро
2. **Ініціалізація** → 3 питання, які встановлюють 4 параметри:
   - `p_alcohol` (0..100) - любов до алкоголю
   - `p_straight` (0..100) - прямолінійність
   - `p_charm` (0..100) - харизма
   - `p_patience` (0..100) - терпіння

3. **Хаб** → меню вибору персонажа:
   - Алекс (легкий/жартівливий) - L2 якщо `p_charm >= 50`
   - Богдан (прагматичний) - L2 якщо `p_patience >= 50`
   - Дана (тусовочна) - L2 якщо `p_alcohol >= 50`
   - Іра (принципова) - L2 якщо `p_straight <= 50`

4. **Після кожної розмови** → повернення в хаб

5. **Статус** → показує, хто погодився піти на пиво

## Важливо

- Ініціалізація налаштована так, щоб **рівно 2 з 4** персонажів мали доступ до L2
- Тільки через L2 можна запросити персонажа на пиво
- Після запрошення встановлюється прапорець `agreed_*`

## Технічні деталі

- Використовує `using game_state` для доступу до змінних
- Умови через `if game_state.параметр`
- Мутації через `set game_state.параметр = значення`
- Переходи через `=> назва_секції`
